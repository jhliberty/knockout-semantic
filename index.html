Where Knockout meets Semantic-UI

What is it?

This is a plugin which facilitates using Semantic-UI and KnockoutJS together.

This project was started when Semantic-UI was just starting out, and Knockout 3.0 was on the horizon.

Knockout works with browsers as old as IE6, but Semantic is a modern browser framework. That gave me the luxury of using the lovely Knockout ES5 plugin, so none of that myObservable(myObservable() + 1) nonsense.

Knockout 3.0 adds some awesome new features, which we take advantage of here.

This isn't for every project, and is designed for new projects.

Download/Install

You can use the official Semantic-UI download builder, which includes the option to use this plugin. Also, bower and git are options.

Make sure things are loaded in the correct order

  1. jQuery
  2. KnockoutJS 3.0 and Semantic-UI
  3. Knockout-ES5
  4. Knockout-Semantic
  5. the script that calls ko.applyBindings()

Toggle

Notice that there are two options for defining elements.

You can use the usual data-bind syntax, or use the custom element syntax. Certain elements support certain bindings, but the data-binding for <sui-something /> will always correspond to the something binding.

Toggle supports the text binding currently.

toggle binding types description
on Boolean

if true the element will get the active class

when clicked the state will toggle

Dropdown

Dropdowns are very useful, and can be thought of as a prettier version of select.

Notice the use of sui.Action in the second dropdown. This lets you set optional callbacks and icons.

If you just want to know when the dropdown changes, make a ko.computed or similar which checks the selected attribute.

dropdown binding types description
data Array[String | Action]

the contents of the dropdown

can be regular strings, or Actions which have a name, callback (optional), and icon (optional)
defaultText String

text to be shown when there is no selection
text String

like defaultText, but doesn't change reguardless of selection
selected String

the most recently chosen item's text

set this to change the selection

Steps

active could be the string "Two", the index 1, or any other value/index. When you set what active is, we'll honor the type when the user changes it. Set "disabled: true" to prevent the user from changing it.

We have a template for this, but you're welcome to use a foreach binding to get a little more control.

steps binding types description
data Array[String] a list of step labels
active Number | String The index or value of the active step, e.g. 1 or "Two"
disabled Boolean true if the user shouldn't be able to change the step by clicking
